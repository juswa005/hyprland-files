# AGENTS.md

## Scope
- This repo is a Hyprland config repo with no README, manifest, CI, or task runner. Treat the config files themselves as the source of truth.

## Live entrypoints
- Main entrypoint: `hyprland.conf`.
- `hyprland.conf` fans out via `source =` into `binds.conf`, `binds-qol.conf`, `inputs.conf`, `windows.conf`, `autostart.conf`, `windowrules.conf`, `hyprsunset.conf`, `monitors.conf`, and `workspaces.conf`.
- `hyprlock.conf` and `hypridle.conf` are separate live configs. They are used by `hyprlock`/`hypridle` launches from `binds.conf` and `autostart.conf`; they are not sourced by `hyprland.conf`.

## What is actually wired
- The active helper-script wiring is centralized at the top of `binds.conf`; if a keybind behavior changes, check the script paths defined there first.
- Session startup behavior lives in `autostart.conf` (`waybar`, `dunst`, `hyprsunset`, `hypridle`, `wl-paste | cliphist`, `swaybg`, `ollama`, `picoclaw`).

## Non-authoritative directories
- `others/` is mostly reference/backup material, not the main live source of truth.
- Do not assume `others/waybar`, `others/kitty`, or `others/dunst` are applied automatically. Repo-local wiring does not point apps at those paths.
- `autostart.conf` starts bare `waybar`, and `scripts/waybar-toggle.sh` also starts/kills bare `waybar`, so repo changes under `others/waybar/` will not take effect unless the user has installed/symlinked them separately.

## Generated files
- `monitors.conf` and `workspaces.conf` are generated by `nwg-displays` and say not to edit manually. Prefer changing monitor/workspace layout through the generating tool, not by hand-editing those files.

## Known gotchas
- `binds.conf` defines `$clipboard = /home/amiel/.config/hypr/scripts/clipboard.sh`, but that script is missing from `scripts/`. Treat it as a stale reference unless the user restores it.
- `windows.conf` declares workspaces `1..10`, while `workspaces.conf` also assigns/defaults workspaces. Be careful not to “fix” workspace behavior in the generated file.
- `scripts/quick-note.sh` creates files under `$HOME/Notes`; it is not a read-only helper.

## Verification
- There is no repo-declared test/lint/build command.
- Verification is manual/runtime. For script-only edits, use focused shell syntax checks such as `bash -n scripts/<name>.sh` before asking the user to exercise the binding/runtime path.
